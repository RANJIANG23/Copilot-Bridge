# Copilot Bridge v1.2.1 界面与可访问性加固设计

> 日期：2026-07-20（Asia/Shanghai）
> 状态：Phase 20–22 已通过；Phase 23 待用户视觉验收与发布候选门禁
> 前置版本：v1.2.0 已发布
> 上位设计：[PROJECT-DESIGN.md](./PROJECT-DESIGN.md)
> 执行路线图：[EXECUTION-ROADMAP.md](./EXECUTION-ROADMAP.md)

## 1. 版本结论

v1.2.1 是一个低风险的桌面体验加固版本。它整理现有 WPF 主题资源，补齐键盘、焦点、触摸目标和辅助技术语义，并完成必要的浅色/深色实机回归；它不改变 v1.2.0 已发布的会话权限、MCP、Edge/CDP/DOM 或发送语义。

这个范围来自两个已核验事实：

- 当前颜色、控件模板和主要版式仍集中在 `MainWindow.xaml`，主题切换依赖同一组 `DynamicResource` 键；
- 项目与模型排序仍以鼠标拖拽为主，部分控件缺少一致的焦点、按下、禁用和自动化名称。

开发期 WorkBuddy/GLM-5.2 记录只作为问题线索。只有本设计明确列出的内容才进入 1.2.1；`.workbuddy/` 继续被 Git 和发布输入排除。

## 2. 必须保持的产品边界

- 继续保持一个生产项目、一个测试项目和一个生产可执行文件。
- 不修改四个 MCP 工具的 schema、注解或权限语义。
- 不修改 Markdown、项目标记、会话分页或授权默认关闭行为。
- 不修改模型选择、页面绑定、单写入锁、发送状态或禁止自动重发语义。
- 不引入第三方 UI 库、MVVM 框架、页面路由框架或第二套主题系统。
- 不拆分 `MainWindow` 为多个 `UserControl`；只提取真正共享的 WPF 资源。
- 不使用 Computer Use、OCR、UIA 或物理输入作为产品运行时能力。

## 3. 交付范围

### 3.1 主题资源与组件状态

- 把颜色、尺寸和共享控件样式移入一个受控的 `ResourceDictionary`，继续使用现有资源键和 `DynamicResource` 主题切换。
- 为常用按钮、输入、选择、列表、菜单和开关统一 `hover`、`pressed`、键盘焦点与禁用状态。
- 只建立当前界面实际使用的 token；不为了形式完整创建未使用的字号、间距或组件层级。
- 保持 Noto Sans SC、Segoe Fluent Icons 和现有浅色/深色视觉方向。

### 3.2 可访问交互

- 关键状态提示使用可被辅助技术识别的名称和温和 live-region 语义。
- 所有主要操作和导航具备明确的自动化名称；纯装饰图标不进入无意义的焦点顺序。
- 项目和模型排序提供键盘替代路径，并继续遵守系统项目锁定与模型候选约束。
- 会话跨项目移动继续使用现有项目选择框与“移动”按钮作为键盘路径，不增加第二套移动逻辑。
- 主要点击目标保持至少约 40 px 高；焦点不因保存、排序或语言切换意外丢失。

### 3.3 布局安全与 Design QA

- 修复当前固定布局在允许的最小窗口和常见 Windows 缩放下出现的截断、重叠或不可达操作。
- 允许必要的换行、滚动或列宽下限调整，但不把 1.2.1 扩大为完整 Adaptive 页面重构。
- 对概览、对话管理、浏览器与模型、设置四个关键页面执行浅色/深色观察；键盘路径必须真实走通。
- 视觉确认由用户观察新启动的 Bridge 实例完成；开发过程不操作 Copilot 页面，也不发送测试消息。

## 4. 明确不做

- onboarding、系统托盘、开机启动、全局热键或自动更新。
- 会话存储 v2、双标题同步、会话级模型控制或新的 MCP 工具。
- 完整响应式导航、移动端式布局或 `MainWindow` 页面组件化重构。
- 为未出现的控件建立完整设计系统或第三方主题兼容层。
- 版本标签、远程推送、GitHub Release 或正式用户目录安装；这些仍需用户明确授权。

## 5. 阶段与门禁

### Phase 20：范围与开发基线冻结

- 新建设计文档并更新总设计、路线图和 README。
- 源码进入 `1.2.1-dev`；已发布下载、Plugin 和发布脚本继续保持 `1.2.0`，避免误发布。
- 通过标准构建、89 项既有测试、依赖/项目数量和 `.workbuddy/` 排除检查。
- 本地提交：`phase 20: freeze v1.2.1 UI hardening scope`。

### Phase 21：主题资源与组件状态

- 提取单一共享主题资源字典，保持现有资源键和主题切换结果。
- 补齐常用控件状态并增加资源完整性测试。
- 浅色/深色不允许出现缺失资源、不可读文本或原有交互回归。
- 本地提交：`phase 21: consolidate WPF theme resources`。

### Phase 22：键盘与辅助技术纵切

- 完成项目/模型排序的键盘路径、自动化名称、live-region、触摸目标和最小布局安全。
- 自动化测试覆盖排序边界、系统项目保护和必要的 XAML 可访问性契约。
- 不发送 Copilot 消息；不修改工作区正文或权限。
- 本地提交：`phase 22: harden accessible desktop interactions`。

### Phase 23：视觉与发布候选门禁

- 运行 Debug/Release 构建、完整测试和生产代码/依赖预算检查。
- 新启动 Bridge 供用户确认浅色、深色、最小窗口与键盘路径。
- 生成候选包、SHA-256 和隔离升级证据只作为门禁；正式安装、标签、推送和 Release 等待用户明确授权。
- 本地提交：`phase 23: harden v1.2.1 release candidate`。

## 6. 复杂度预算与停止条件

- 生产代码保持在总体约 7000 行以内；1.2.1 生产增量目标不超过约 650 行。
- 直接生产 NuGet 依赖保持 2 个，不新增 UI 包。
- 最多新增一个共享主题资源字典和少量直接支撑测试；不建立 token 生成器或主题框架。
- 如果提取资源导致主题切换、语言切换或主要页面行为回归，Phase 21 停止在原门禁修复，不能并行保留两套样式。
- 如果键盘替代要求复制排序/移动业务逻辑，先收敛为同一现有处理链，不能增加第二个状态源。
