## 2. v1.1.2 纳入标准

一个任务只有同时满足以下条件，才属于 v1.1.2：

- 不增加或修改 MCP 工具面和公开调用契约。
- 不改变 Markdown 会话格式、稳定 ID、目录结构或文件命名规则。
- 不新增后台服务、进程间协议、数据库、队列、浏览器栈或生产依赖。
- 不改变现有 Edge CDP/DOM、单击一次发送和不确定状态零重发边界。
- 不引入需要用户重新决定的产品模型，例如内容交接、项目级调用或自动模式路由。
- 可以在一个现有模块或一个现有页面内完成，并能用自动化测试或明确的人工观察验收。
- 不突破一个生产项目、一个测试项目和一个生产可执行文件的结构。

### 3.2 Phase 14：概览状态静默刷新

#### 问题

自动状态刷新当前会调用全局 `SetBusy(true/false)`，周期性禁用界面并改变状态文案，造成概览页可见闪烁。定时刷新不应表现为一次前台用户操作。

#### 实现边界

- 自动刷新不调用全局 `SetBusy`，不禁用导航、按钮和输入控件。
- 增加独立的刷新中标志，阻止自动刷新、手动刷新和咨询过程发生重复状态读取。
- 刷新期间保留上一次成功状态；读取完成后只更新发生变化的字段。
- 自动刷新失败继续使用已有 30/60/120 秒退避，并以非阻塞状态提示表达。
- 首次尚无状态时可以显示静态加载提示；十秒周期刷新不得反复显示骨架屏。
- 不改变 Edge 页面、不发送 Copilot 消息，也不增加自动重试发送。

#### 验收

- 自动刷新期间导航和普通按钮保持可用。
- 连续观察至少 60 秒，没有全页禁用、状态条抖动或周期性骨架屏。
- 自动刷新与手动刷新不会并发执行。
- 咨询正在进行时保持现有跳过刷新语义。
- 现有刷新间隔和退避测试继续通过，并补充并发保护测试。

### 3.3 两项低风险桌面入口调整

这两项只调整现有页面入口，不改变业务能力：

1. 将“即时咨询”主要入口从设置页移动到概览页；设置页只保留配置。即时咨询仍调用现有同一处理链，不新增发送路径。
2. 在“本地会话工作区”路径控件旁增加“打开”操作，用系统文件管理器打开当前工作区；现有“浏览”仍只负责选择目录。

验收：

- 概览页即时咨询与原入口使用同一验证、忙碌保护、保存和错误处理代码。
- 设置页不再重复展示同一个主要业务操作。
- “打开”只接受已解析的当前工作区目录，不拼接 shell 命令；目录不存在时给出明确错误。
- 不改变 Markdown 文件、项目分类和导入行为。

### 3.4 Noto Sans SC 分发闭环

当前 XAML 指定了 `Noto Sans SC`，但项目尚未把字体作为发布资源携带。开发机已安装字体不能证明安装包在其他机器上也会使用该字体。

v1.1.2 必须在发布前完成以下二选一，并以用户要求的 Noto Sans SC 为优先：

- 从可核验的官方来源引入允许再分发的 Noto Sans SC 字体资源，同时在发布物中保留对应许可证/NOTICE，并验证 WPF 使用打包字体；或
- 如果许可证、体积或 WPF 打包验证未通过，则停止发布并报告该门禁，不把系统回退字体描述为已完成替换。

不得从 WorkBuddy 缓存、临时目录或来源不明的本机字体文件复制发布资源。
